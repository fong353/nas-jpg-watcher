# NAS JPG 元数据巡检 — Win 端部署

## 用途

在 Win 电脑上定时轮询 NAS 共享目录，**只修 JFIF:ResolutionUnit = None 的 JPG**（把单位位从"无"翻成"inches"），其他文件完全不动。ErgoSoft / PS / 任何下游软件读到的尺寸立即正确。

- **无损**：不重压缩像素流，不改数值，只翻 1 字节的枚举值
- **幂等**：已经正常的文件 `-if` 条件直接跳过，0 写入、0 网络流量、SHA 不变
- **非侵入**：`-P` 保持原文件修改时间，NAS 上其他同步工具不会误判

## 一、文件清单（本目录）

| 文件 / 目录 | 作用 |
|---|---|
| `scan.cmd` | 手动调试入口（双击可见窗口输出）。日常被 `scan-silent.vbs` 静默调用 |
| `scan-silent.vbs` | 定时任务真正的入口，隐藏 scan.cmd 窗口，不闪黑框 |
| `scan.ps1` | **真正的扫描逻辑 & 配置文件**，改 NAS 路径和扫描天数都在这里 |
| `exiftool/` | ExifTool 13.57 Win 64 bit 绿色版，自带，不用另装 |
| `install-task.cmd` | 一键注册"每 30 分钟跑一次"到 Windows 任务计划程序 |
| `uninstall-task.cmd` | 一键卸载该任务 |
| `scan.log` | 日志文件（首次运行后生成） |
| `部署说明.md` | 本文件 |

## 二、部署步骤（5 分钟）

### 1. 把整个文件夹拷到 Win 电脑

推荐 `C:\Tools\nas-jpg-watcher\`（位置固定就行，任务里写的是绝对路径）。

### 2. 改 scan.ps1 里的配置

用记事本或 VS Code 打开 **`scan.ps1`**（不是 `scan.cmd`），顶部配置区：

```powershell
$Roots = @(
    '\\192.168.0.150\nas\小红书\2026',
    '\\192.168.0.150\nas\淘宝\2026'
)
$Days = 5
```

- **`$Roots`**：**父目录**列表。脚本会列出它下面的一级子目录，**只保留最近 `$Days` 天有内容变动的那些**（按 `LastWriteTime` 筛）——文件夹被加/改/删文件都会触发。和子目录叫什么名字无关。加目录直接在数组里加一行（上一行末尾留好逗号）。
  - **必须用 UNC 路径**（`\\IP\共享名\子目录`）。盘符（`Z:\`）后台任务拿不到映射。
- **`$Days`**：保留最近 N 天有变动的子目录。默认 5。想更激进改 1-2，保守改 10-30。

**关键前置**：先在 Win 资源管理器里用同一账号能**无密码**打开该路径。如要认证，Win+R → `control keymgr.dll` → 凭据管理器存好账密。

### 3. 手动跑一次验证

双击 `scan.cmd`，立即执行一次。完成后看 `scan.log`：

```
[2026-04-23 12:30:01] --- scan start (last 5 d) ---
[2026-04-23 12:30:01]     target: \\192.168.0.150\nas\小红书\2026\4月  (modified 2026-04-23 12:29)
[2026-04-23 12:30:01]     target: \\192.168.0.150\nas\淘宝\2026\4月  (modified 2026-04-23 11:15)
    1 image files updated
[2026-04-23 12:30:04] --- scan done ---
```

判断：
- 看到一行或多行 `target: ...  (created ...)` = 找到最近 N 天创建的子目录，**确认时间合理**
- 看到 `N image files updated` = 修了 N 张
- 只有 start/done + target 没别的 = 没需要修的，一切正常
- 看到 `WARN: root not accessible` = UNC 路径或权限问题，在资源管理器里试试能不能打开
- 看到 `Error:` = 按错误信息处理（通常是权限 / 文件被占用）

### 4. 注册定时任务（每 30 分钟）

**右键 `install-task.cmd` → 以管理员身份运行**。成功后可在 "任务计划程序 → 任务计划程序库" 看到 `NAS JPG Watcher`。

之后只要 Win 电脑开机、用户登录，它就每 30 分钟自动扫一次。

### 5. 卸载

双击 `uninstall-task.cmd`（管理员权限不是必须）。

## 三、扫描周期 & 性能说明

- 默认每 30 分钟一轮。要更实时改小，不急改大。HDD NAS 上一轮大约几十秒（按当日活跃目录数量走）
- **按 LastWriteTime 两层筛选子目录**：每轮只进"最近 N 天有变动"的月→日目录，历史完全跳过
- `-if` 过滤：已经正常的文件只读 JFIF 头不写盘，对 NAS 压力极小

**改周期**：编辑 `install-task.cmd`，改 `/sc minute /mo 30` 这一行（`/mo` 后面的数字 = 间隔）。常用组合：

| 想要 | 改成 |
|---|---|
| 每 5 分钟 | `/sc minute /mo 5` |
| 每 10 分钟 | `/sc minute /mo 10` |
| 每 30 分钟（默认） | `/sc minute /mo 30` |
| 每小时 | `/sc hourly /mo 1` |
| 每 2 小时 | `/sc hourly /mo 2` |

改完 **先双击 `uninstall-task.cmd` 卸旧任务**，再右键 `install-task.cmd` → 以管理员身份运行。任务的周期在注册时固化，不重装不生效。

## 四、文件数量较多时的优化（>10,000 张 JPG）

当前版本已经按子目录创建时间切片，历史数据不会重复扫描，一般不会遇到瓶颈。如果某天日增量爆到每天 5000+ 张，可考虑：

1. **只扫最近 10 分钟改过的文件**：在 `scan.ps1` 里加 mtime 预过滤，把新文件列表传给 exiftool
2. **并行扫两个根**：PowerShell 里用 `Start-Job` 并行跑小红书和淘宝

需要时告诉我文件量和目录结构，我调整 `scan.ps1`。

## 五、问题排查

### 日志里 `Error: File not found`

UNC 路径错误或 NAS 未连。在资源管理器里试一下能不能打开。

### 日志里 `Error: Can't open file for writing`

该文件正在被上传或被其他程序占用。脚本自动跳过，下一轮会再试。**不需要处理**。

### 某张 JPG 怎么都没被修

说明它不是故障模式 1。诊断命令（CMD 里跑）：

```
"C:\Tools\nas-jpg-watcher\exiftool\exiftool.exe" -G1 -a -s -ResolutionUnit -XResolution -YResolution "问题图.jpg"
```

把输出贴给我，按 5 种故障模式（参考主 plan 文件）判断。

### 任务计划程序显示"上次运行结果 0x1"

脚本本身没问题，可能是权限不足（读不到 NAS）。确认：
- 任务属性 → 常规 → "不管用户是否登录都运行" 勾上
- 填存了 NAS 凭据的 Win 账号密码
- 或者改成 "只在用户登录时运行"（简单可靠）

## 六、备份

整个 `nas-jpg-watcher\` 文件夹拷走即可，30 MB 出头。换机器：复制 + 运行 `install-task.cmd`。
